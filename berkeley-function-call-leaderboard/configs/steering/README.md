# Activation Steering Configs for BFCL

This directory contains YAML configs for running BFCL inference with activation steering.
Steering applies pre-computed intervention vectors to transformer layers at generation time
to study how targeted interventions affect function-calling parameter quality.

---

## Prerequisites

### 1. Compute steering vectors first

Steering vectors must be produced by the `activation_analysis` project **before** running BFCL
with steering. They are NOT generated by BFCL itself.

```bash
cd /home/ae0589/project/toe-param-analysis/activation_analysis
# run compute_steering_vectors with your config
python -m steering.compute_steering_vectors \
    --config configs/steering/compute_steering_vectors.sample.yaml
```

The output is a `steering_vectors.pt` file (a dict mapping layer names to tensors).
Note the directory path — you will set it in the BFCL steering config.

### 2. Update the config path

Open the relevant YAML in this directory and set `steering.source.path` to the directory
containing your `steering_vectors.pt`:

```yaml
steering:
  source:
    type: path
    path: /absolute/path/to/dir_with_steering_vectors_pt   # <-- edit this
```

## Running steered inference

Run from the `berkeley-function-call-leaderboard` root:

```bash
bfcl generate \
  --model qwen3-8b-FC-steered \
  --steering-config configs/steering/bfcl_qwen3_8b.yaml \
  --test-category simple_python \
  --allow-overwrite
```

Then evaluate:

```bash
bfcl evaluate \
  --model qwen3-8b-FC-steered \
  --test-category simple_python
```

To compare against the unsteered baseline, run the same commands with `qwen3-8b-FC`
(no `--steering-config` flag, uses vLLM server instead of direct HF inference).

## Config schema

```yaml
model:
  name: "Qwen/Qwen3-8B"     # HuggingFace model ID (or local path via --local-model-path)
  dtype: "bfloat16"          # bfloat16 | float16 | float32

sampling:
  enable_thinking: true      # true for models with <think> tokens (Qwen3), false otherwise
  max_new_tokens: 4096
  temperature: 0.001         # near-greedy recommended for reproducibility
  top_p: 1.0
  top_k: 1
  repetition_penalty: 1.0
  do_sample: false
  thinking_tokens:           # only needed if enable_thinking: true
    start: 151667            # token ID for <think>
    end: 151668              # token ID for </think>

steering:
  source:
    type: path               # 'path' = directory containing steering_vectors.pt
    path: /abs/path/to/dir   # OR use type: file with path to .pt file directly
  steering_sets:
    - token_span: answer_mean   # name of the vector key inside steering_vectors.pt
      coef: -1.0                # scaling coefficient (negative = suppress direction)
      apply_target: answer      # where to apply: all | reasoning | answer
      layers:
        - layer25               # layer index (e.g. layer0..layer31 for 32-layer model)
      max_tokens: null          # max tokens to apply steering for; null = no limit
```

### `apply_target` values

| Value | When to use |
|-------|-------------|
| `all` | No thinking phase (Llama, Mistral) — steer all tokens |
| `reasoning` | Steer only the `<think>...</think>` span (Qwen3) |
| `answer` | Steer only the answer span after `</think>` (Qwen3) — most common |

### Multiple steering sets

You can stack multiple interventions in one run:

```yaml
steering:
  source:
    type: path
    path: /path/to/vectors
  steering_sets:
    - token_span: answer_mean
      coef: -1.0
      apply_target: answer
      layers: [layer25]
    - token_span: reasoning_mean
      coef: 0.5
      apply_target: reasoning
      layers: [layer20, layer25]
```

## How it works (architecture overview)

```
bfcl generate --steering-config ...
        │
        ▼
_llm_response_generation.py
  - detects SteeringMixin handler
  - sets num_threads=1 (model.generate() not thread-safe)
  - skips vLLM server startup
  - calls handler.load_hf_model(model_path, steering_config)
        │
        ▼
SteeringMixin.load_hf_model()
  - loads HF model + tokenizer onto GPU
  - auto-detects thinking token IDs from tokenizer
  - loads steering_vectors.pt
  - builds steering_sets list
        │
        ▼
SteeringMixin._steered_generate()  [called per sample]
  - formats prompt via parent handler (_format_prompt)
  - attaches activation hooks to specified layers
  - calls run_generation() from steering_primitives.py
  - hooks steer activations during forward pass
  - reconstructs <think>...</think>\n\nanswer text
  - returns FakeCompletionResponse (duck-types OpenAI response)
        │
        ▼
BFCL evaluation pipeline (unchanged)
  - AST parsing, type/value matching, param_level_stats
```

The evaluation pipeline is completely unaware of steering — it receives the same response
format as a normal vLLM run.

## Troubleshooting

**`FileNotFoundError: Steering vectors file not found`**
→ `steering.source.path` in your YAML points to the wrong directory, or vectors have not
been computed yet. Run `compute_steering_vectors` in `activation_analysis` first.

**`[steering] Ready.` never appears / hangs after `Max context length`**
→ Already fixed: was a polling bug in `spin_up_local_server`. Make sure you have the latest code.

**`No such option: --steering-config`**
→ Reinstall in editable mode: `pip install -e . --no-deps` from the project root.

**Model loads but output looks wrong / no function calls**
→ Check `apply_target` — for Qwen3 answer steering use `answer`, not `all`.
→ Check `coef` sign — a large positive coefficient can push the model off-distribution.
→ Try `coef: 0.0` first as a sanity check (should match unsteered output).
